# MediaGenerator - Roadmap

Каждая фаза — независимый блок, который можно выполнить в отдельной сессии.
После каждой фазы проект должен быть в рабочем состоянии.

---

## Фаза 1: Фундамент

> Цель: запустить пустое приложение в Docker с БД, авторизацией и базовым UI.

### Задачи

- [ ] **1.1** Настроить Next.js 15 + TypeScript + Tailwind 4 + shadcn/ui
  - Тёмная тема по умолчанию
  - Русскоязычный интерфейс
  - Базовый `layout.tsx` с провайдерами

- [ ] **1.2** Создать Docker Compose
  - PostgreSQL 16 (volume для данных)
  - MinIO (volume для изображений, web-консоль на :9001)
  - Next.js app (standalone build)
  - `.env.example` с описанием всех переменных

- [ ] **1.3** Настроить Drizzle ORM
  - Подключение к PostgreSQL
  - Все таблицы из схемы БД (см. ARCHITECTURE.md)
  - Первая миграция
  - `drizzle.config.ts`

- [ ] **1.4** Настроить Better Auth
  - Email + пароль аутентификация
  - Роли: `admin` / `user`
  - Первый пользователь автоматически = admin
  - Страницы: логин, регистрация
  - Middleware для защиты роутов

- [ ] **1.5** Настроить MinIO S3 клиент
  - `@aws-sdk/client-s3` (совместим с MinIO)
  - Функции: upload, download, delete, getSignedUrl
  - Авто-создание бакета при старте

- [ ] **1.6** Базовый layout дашборда
  - Sidebar с навигацией (Генерация, Библиотека, История, Настройки)
  - Header с именем пользователя и выходом
  - Пустые страницы-заглушки для всех разделов

### Проверка
```
docker compose up → все сервисы стартуют
localhost:3000 → редирект на логин
Регистрация → вход → dashboard с sidebar
MinIO console на localhost:9001
```

## Фаза 2: Генерация изображений

> Цель: генерировать изображения через все 3 провайдера и видеть результаты в вебе.

### Задачи

- [ ] **2.1** Адаптеры провайдеров
  - OpenAI: все 3 модели, все параметры
  - xAI: обе модели, все параметры
  - OpenRouter: 3-5 топовых моделей, параметры
  - Общий интерфейс `ImageProvider`
  - Обработка ошибок, retry, таймауты

- [ ] **2.2** Реестр моделей
  - Таблица `model_registry` с предзаполнением
  - API для получения моделей с группировкой по провайдерам
  - Схемы параметров для каждой модели (для динамической генерации UI)

- [ ] **2.3** Страница настроек: API ключи
  - Форма ввода ключей для каждого провайдера
  - AES-256 шифрование
  - Показ маскированного ключа (***xF4k)
  - Проверка валидности ключа при сохранении

- [ ] **2.4** Страница генерации: UI
  - Селектор провайдера и модели
  - Динамическая панель параметров (из params_schema)
  - Textarea для промпта
  - Выбор количества изображений
  - Кнопка "Сгенерировать"
  - Сохранение пользовательских дефолтов (user_preferences)

- [ ] **2.5** Логика генерации
  - API route `POST /api/generate`
  - Проверка лимита пользователя
  - Запись в `generations` (pending → processing → done/error)
  - Вызов адаптера провайдера
  - Загрузка результатов в MinIO
  - Запись в `images`
  - Расчёт стоимости

- [ ] **2.6** Отображение результатов
  - Сетка сгенерированных изображений
  - Скелетоны / спиннер во время генерации
  - Лайтбокс: клик → полноэкранный просмотр с затемнением
  - Кнопка скачивания в лайтбоксе

### Проверка
```
Настройки → ввод API ключа OpenAI → сохранение → маскированное отображение
Генерация → выбор gpt-image-1-mini → промпт → "Сгенерировать"
Результат появляется в сетке
Клик → лайтбокс → скачивание работает
Повторить для xAI и OpenRouter
MinIO console → изображения в бакете
```

## Фаза 3: Библиотека и история

> Цель: все изображения организованы в папки, есть полная история генераций.

### Задачи

- [ ] **3.1** Библиотека: сетка изображений
  - Адаптивная сетка с превью
  - Ленивая загрузка (intersection observer)
  - Мультивыбор (чекбоксы)
  - Контекстное меню (удалить, переместить, скачать)

- [ ] **3.2** Библиотека: папки
  - Дерево папок в левой панели
  - Создание, переименование, удаление папок
  - Drag & drop: перетаскивание изображений между папками
  - Вложенные папки (parent_id)

- [ ] **3.3** Библиотека: массовые операции
  - Выделить все / снять выделение
  - Массовое удаление
  - Массовое перемещение в папку
  - Массовое скачивание (zip-архив)

- [ ] **3.4** История генераций
  - Таблица с фильтрами: по дате, модели, провайдеру, статусу
  - Пагинация
  - Клик по записи → детали (промпт, параметры, изображения)
  - Повторная генерация с теми же параметрами

### Проверка
```
Библиотека → все изображения отображаются
Создание папки → перетаскивание изображений → отображение в папке
Мультивыбор → удаление → изображения удалены из MinIO и БД
История → фильтр по дате → детали генерации → "Повторить"
```

## Фаза 4: Админка и уведомления

> Цель: управление пользователями, API ключами и автоматические уведомления.

### Задачи

- [ ] **4.1** Управление пользователями
  - Таблица пользователей (только для admin)
  - Изменение роли (admin/user)
  - Установка дневного лимита генераций
  - Блокировка (лимит = 0)

- [ ] **4.2** Проверка обновлений моделей (cron)
  - node-cron: запуск раз в день (06:00 UTC)
  - Запрос списка моделей у каждого провайдера
  - Сравнение с model_registry
  - Создание уведомлений о новых/удалённых моделях
  - Автодобавление новых моделей с is_active: false (админ активирует вручную)

- [ ] **4.3** Страница уведомлений
  - Список уведомлений с типом и датой
  - Индикатор непрочитанных в sidebar
  - Отметка "прочитано"
  - Действия: активировать новую модель

### Проверка
```
Admin → Пользователи → изменить лимит → пользователь видит ограничение
Ручной триггер проверки моделей → появляется уведомление
Уведомления → "Активировать модель" → модель появляется в селекторе
```

## Фаза 5: Полировка

> Цель: продакшн-качество, производительность, удобство.

### Задачи

- [ ] **5.1** Производительность
  - Виртуализация длинных списков (react-virtuoso)
  - Lazy loading изображений
  - Оптимизация запросов к БД (индексы)
  - Image optimization (Next.js Image)

- [ ] **5.2** UX-полировка
  - Анимации переходов (framer-motion)
  - Горячие клавиши (Ctrl+Enter для генерации)
  - Toast-уведомления о результатах
  - Пустые состояния (empty states)

- [ ] **5.3** Обработка ошибок
  - Graceful degradation при недоступности провайдера
  - Retry с exponential backoff
  - Понятные сообщения об ошибках на русском
  - Error boundaries для React компонентов

- [ ] **5.4** Responsive design
  - Мобильная версия sidebar (sheet/drawer)
  - Адаптивная сетка изображений
  - Тачскрин-совместимый drag & drop

- [ ] **5.5** Документация деплоя
  - README с инструкцией установки
  - Настройка Cloudflare Tunnel
  - Бэкап PostgreSQL и MinIO
  - Обновление приложения

### Проверка
```
Lighthouse score > 90
Нет ошибок в консоли
Мобильная версия работает
docker compose up на чистой машине → всё работает
```
